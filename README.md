# Razreshenie VPN Client

**Razreshenie VPN Client** — открытый VPN-клиент на Python для Windows.

**Версия:** `1.1.5`

Слоган: **«Разреши себе доступ к любым сайтам»**.

Проект распространяется как свободное ПО под лицензией **GPLv3**. Код открыт, телеметрии нет, скрытых функций нет, сбор пользовательских данных не выполняется. Профили, подписки, правила и настройки хранятся локально.

Репозиторий проекта: <https://github.com/communism420/Razreshenie-VPN-Client>

## Возможности

- GUI на PyQt6 + PyQt6-Fluent-Widgets с темной Fluent-темой в стиле zapret-kvn.
- VLESS import из одиночных `vless://` ключей с Karing-style определением названий серверов.
- Emoji в названиях серверов сохраняются, а флаги стран отображаются как настоящие SVG-иконки вместо буквенных региональных индикаторов Windows.
- Импорт подписок в base64, plain-text, JSON со строками `vless://` или JSON/sing-box/Clash outbound-объектами VLESS.
- Устойчивое обновление подписок: клиент объединяет несколько ответов провайдера и не удаляет уже найденные серверы при частичном ответе.
- Karing-style проверка отклика выбранного сервера или всех серверов сразу: временный sing-box core поднимает outbounds и меряет задержку через Clash API `/proxies/{tag}/delay`, без системного `ping.exe` и без блокировки GUI.
- Карточка `Общий трафик` на панели показывает, сколько данных принято и отправлено за текущую VPN-сессию.
- График скорости трафика анимируется как плавная лента: новая часть аккуратно выезжает справа, а история уходит влево без деформации линий.
- sing-box core с автоматической загрузкой последнего Windows x64 release и проверкой SHA256, если checksum есть в release.
- Proxy-режим через mixed inbound SOCKS5/HTTP.
- TUN-режим с virtual adapter, auto route, Karing-style Windows TUN stack/MTU и strict route при включенном Kill Switch.
- Karing-style подключение: preflight-проверки профиля и сервера, контроль занятого proxy-порта, ожидание реальной готовности inbound/TUN, outbound health-check через запущенный core и повтор запуска с cleanup.
- Split tunneling через локальные JSON/TXT-файлы или raw GitHub URL.
- Встроенный bypass с доменами из российского «белого списка».
- Для каждого JSON/TXT-набора отдельно выбирается маршрут: через текущий сервер или напрямую.
- Live-окно активности доменов: видно, какие текущие домены и поддомены идут через VPN, а какие напрямую, с фильтром по словам и легкой виртуальной таблицей без зависаний при пролистывании.
- Логи, экспорт логов, DNS-проверка, системный трей.
- Автообновление подписок, автоподключение, автозапуск Windows, proxy guard и Windows-уведомления.
- Кнопка полного сброса удаляет настройки, серверы, подписки, правила, runtime-конфиги и скачанный sing-box core, после чего закрывает приложение.
- Все пользовательские данные, настройки, профили, правила, логи и sing-box core хранятся в `%USERPROFILE%\Razreshenie VPN`.

## Быстрый запуск из исходников

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
python main.py
```

Proxy-режим работает без прав администратора. TUN-режим при включении или подключении автоматически запросит UAC, если у процесса еще нет прав администратора.

## Проверка кода

```powershell
python -m compileall -q .
python main.py --self-check
```

`--self-check` проверяет парсер VLESS и генерацию базового sing-box config без запуска GUI.

## Подписки и серверы

Во вкладке `Подписки` можно добавить URL подписки, обновить выбранную подписку или все подписки сразу. При обновлении клиент сначала делает быстрый основной запрос; дополнительные параллельные загрузки включаются только если ответ выглядит неполным. Локальный список серверов этой подписки заменяется актуальным списком провайдера, а счетчик профилей считается по фактическому составу группы, как в Karing: повторяющиеся VLESS-записи не схлопываются и остаются отдельными серверами. Если у нескольких серверов одинаковое название, повторные записи получают суффиксы `-1`, `-2` и далее.

Во вкладке `Серверы` профили сгруппированы по подпискам: категорию подписки можно свернуть или развернуть кликом по ее строке. Там же можно импортировать профиль, активировать сервер одним кликом по строке, открыть JSON профиля, проверить сгенерированный sing-box config, удалить сервер, проверить отклик выбранного сервера или всех серверов через Karing-style URL-test и отсортировать таблицу по отклику.

## Маршрутизация JSON/TXT

Для JSON поддерживаются структуры с ключами:

- `domains`, `domain`, `domain_suffix`, `domain_keyword`
- `ip`, `ips`, `ip_cidr`, `cidr`
- `process_name`, `process_path_regex`

Пример:

```json
{
  "domains": ["example.com", "*.example.org"],
  "ip_cidr": ["1.1.1.1/32", "8.8.8.0/24"]
}
```

Для TXT поддерживается простой список: один домен или IP/CIDR на строку. Пустые строки и строки с `#`, `;`, `//` пропускаются.

```text
wildberries.ru
wb.ru
*.example.com
1.1.1.0/24
```

Можно загрузить несколько локальных JSON/TXT-файлов или прямых raw-ссылок GitHub, например:

```text
https://raw.githubusercontent.com/communism420/My-Karing-Ruleset/refs/heads/main/griffini.json
```

Для каждого набора в интерфейсе отдельно задается маршрут:

- `Текущий сервер` — совпавший трафик идет через активный VLESS-профиль.
- `Напрямую` — совпавший трафик идет в обход VPN.

После добавления набора настройте его маршрут прямо в строке таблицы в колонке `Туннелирование`. Если включены только правила `Напрямую`, весь остальной трафик идет через текущий сервер. Если включены только правила `Текущий сервер`, весь остальной трафик идет напрямую. При смешанных правилах используется внутренний маршрут по умолчанию.

Встроенный bypass всегда добавляется к runtime-конфигу sing-box и отправляет домены российского «белого списка» напрямую. Пользовательские правила имеют более высокий приоритет, чем встроенный bypass.

## Активность доменов

Вкладка `Активность` показывает свежие домены и поддомены, которые `sing-box` упоминает в runtime-логах. Для каждой записи отображается маршрут: `VPN` или `Напрямую`, правило, количество запросов и время последнего появления.

Фильтр принимает одно или несколько слов: например `google video` покажет только домены, где встречаются оба слова.

## Сборка в один EXE через PyInstaller

1. Установите зависимости:

```powershell
python -m pip install -r requirements.txt
```

2. Создайте `.ico`:

```powershell
python tools\create_icon.py
```

3. Соберите один EXE без консоли и с иконкой:

```powershell
pyinstaller --noconfirm --clean --onefile --windowed `
  --name "Razreshenie VPN Client" `
  --icon assets\app.ico `
  --version-file tools\windows_version_info.txt `
  --collect-data qfluentwidgets `
  --hidden-import qfluentwidgets `
  --add-data "logo.webp;." `
  --add-data "assets;assets" `
  --distpath exe_release `
  main.py
```

EXE появится в папке `exe_release`.

Сборка выше не требует права администратора при обычном запуске. TUN-режим запросит UAC только при включении или подключении, если прав администратора еще нет.

- отсутствие телеметрии;
- отсутствие скрытых сетевых запросов, кроме запросов, инициированных пользователем: загрузка sing-box, подписок, raw JSON/TXT-правил, проверки IP/DNS;
- хранение пользовательских данных локально;
- воспроизводимая сборка из исходников.
