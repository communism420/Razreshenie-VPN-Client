# Razreshenie VPN Client

**Razreshenie VPN Client** — открытый VPN-клиент на Python для Windows.

**Версия:** `2.0.1`

Слоган: **«Разреши себе доступ к любым сайтам»**.

Проект распространяется как свободное ПО под лицензией **GPLv3**. Код открыт, телеметрии нет, скрытых функций нет, сбор пользовательских данных не выполняется. Профили, подписки, правила и настройки хранятся локально.

Репозиторий проекта: <https://github.com/communism420/Razreshenie-VPN-Client>

## Возможности

- GUI на PyQt6 + PyQt6-Fluent-Widgets с темной Fluent-темой в стиле zapret-kvn.
- Импорт одиночных URI для VLESS, Trojan, VMess, Hysteria2, TUIC v5, Shadowsocks и WireGuard.
- Emoji в названиях серверов сохраняются, а флаги стран отображаются как настоящие SVG-иконки вместо буквенных региональных индикаторов Windows.
- Импорт подписок в форматах base64/plain-text, Clash YAML/JSON, sing-box JSON, v2rayN/Nekoray JSON и списков URI.
- Karing-style разбор названий серверов, групп и тегов из подписок.
- Устойчивое обновление подписок: клиент делает быстрый основной запрос, при подозрении на неполный ответ запускает дополнительные параллельные запросы и применяет самый полный полученный снимок подписки.
- Импорт нескольких ссылок или файлов за один проход с прогрессом.
- Karing-style проверка отклика выбранного сервера или всех серверов сразу: временный sing-box core поднимает outbounds и измеряет задержку через Clash API `/proxies/{tag}/delay`, использует Karing URL-test список с быстрым fallback, без системного `ping.exe` и без блокировки GUI.
- Smart Connect: при подключении клиент выбирает лучший сервер из текущей подписки/группы по задержке и накопленной статистике качества; при необходимости это можно отключить в настройках.
- Failover-группы: при падении текущего сервера клиент может автоматически переключиться на следующий живой сервер из группы.
- История качества серверов: задержка, успешные и неуспешные проверки сохраняются локально.
- Карточка `Общий трафик` на панели показывает, сколько данных принято и отправлено за текущую VPN-сессию.
- График скорости трафика анимируется как плавная лента: новая часть аккуратно выезжает справа, а история уходит влево без деформации линий.
- sing-box core с автоматической загрузкой последнего Windows x64 release и проверкой SHA256, если checksum есть в release.
- Проверка и безопасная загрузка обновлений самого приложения через GitHub Releases. Автоматическая замена запущенного exe не выполняется.
- Proxy-режим через mixed inbound SOCKS5/HTTP.
- TUN-режим с virtual adapter, auto route, IPv4/IPv6 адресами, Karing-style Windows TUN stack/MTU и strict route при включенном Kill Switch.
- DNS strategy для IPv4/IPv6: prefer IPv4, prefer IPv6, IPv4 only, IPv6 only.
- Karing-style подключение: быстрый старт sing-box без блокирующего outbound health-check, контроль занятого proxy-порта, короткое ожидание готовности локального inbound/core и фоновая проверка выхода через текущий сервер. В TUN-режиме отсутствие IPv4 в Windows не блокирует статус подключения.
- Split tunneling через локальные JSON/TXT/SRS-файлы или raw URL.
- Встроенный bypass с доменами из российского «белого списка».
- Для каждого JSON/TXT/SRS-набора отдельно выбирается маршрут: через текущий сервер или напрямую, с управлением приоритетами.
- Live-окно активности доменов: видно, какие текущие домены и поддомены идут через VPN, а какие напрямую, с фильтрами, статистикой и быстрым созданием правил.
- Per-app routing через `process_name`, `process_path` и `process_path_regex`.
- Kill Switch для TUN и отдельный opt-in Firewall Kill Switch Windows.
- Self-healing: автоматическое восстановление sing-box при неожиданном падении с ограничением попыток и cooldown.
- Диагностика ZIP: экспорт логов, redacted-конфигов, настроек и системной информации для баг-репортов без секретов.
- Логи, DNS-проверка, системный трей.
- Автообновление подписок, автоподключение, автозапуск Windows, proxy guard и Windows-уведомления.
- Настройки применяются автоматически сразу после изменения; отдельной кнопки сохранения нет.
- Кнопка полного сброса удаляет настройки, серверы, подписки, правила, runtime-конфиги и скачанный sing-box core, после чего закрывает приложение.
- Все пользовательские данные, настройки, профили, правила, логи и sing-box core хранятся в `%USERPROFILE%\Razreshenie VPN`.

## Быстрый запуск из исходников

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
python main.py
```

Proxy-режим работает без прав администратора. TUN-режим при включении или подключении автоматически запросит UAC, если у процесса еще нет прав администратора.

## Проверка кода

```powershell
python -m compileall -q .
python main.py --self-check
```

`--self-check` проверяет парсеры серверов, генерацию sing-box config, маршрутизацию, Smart Connect, диагностику, updater-логику и обработку ошибок без запуска GUI.

## Поддерживаемые протоколы

Один и тот же профиль работает в Proxy-режиме и TUN-режиме: меняется только inbound sing-box, outbound собирается одинаково.

| Протокол | Импорт | Примечания |
| --- | --- | --- |
| VLESS | `vless://`, sing-box/Clash/Nekoray JSON | TLS, Reality, transports, flow, mux |
| Trojan | `trojan://`, sing-box/Clash/Nekoray JSON | TLS и Reality |
| VMess | `vmess://`, JSON | TCP/WS/gRPC/HTTP transports, TLS |
| Hysteria2 | `hysteria2://`, `hy2://`, JSON | QUIC/UDP outbound, obfs salamander |
| TUIC v5 | `tuic://`, JSON | UUID/password, congestion control |
| Shadowsocks | `ss://`, JSON | Включая методы Shadowsocks 2022 |
| WireGuard | `wireguard://`, sing-box JSON | Опционально; нужен корректный набор ключей и адресов |

## Подписки и серверы

Во вкладке `Подписки` можно добавить URL подписки, обновить выбранную подписку или все подписки сразу. Поддерживаются обычные base64/plain-text подписки, Clash YAML/JSON, sing-box JSON, v2rayN/Nekoray JSON и списки URI. При обновлении клиент сначала делает быстрый основной запрос; дополнительные параллельные загрузки включаются только если ответ выглядит неполным. Из полученных ответов используется самый полный снимок, после чего локальный список серверов этой подписки заменяется актуальным списком провайдера.

Счетчик профилей считается по фактическому составу группы, как в Karing: повторяющиеся записи не схлопываются и остаются отдельными серверами. Если у нескольких серверов одинаковое название, повторные записи получают суффиксы `-1`, `-2` и далее. Названия, группы, теги, флаги и типы серверов нормализуются при импорте, чтобы в таблице было видно, чем VLESS, Trojan, VMess, HY2, TUIC, SS и WG отличаются друг от друга.

Во вкладке `Серверы` профили сгруппированы по подпискам и группам провайдера: категорию можно свернуть или развернуть кликом по ее строке. Там же можно импортировать один или несколько профилей, активировать сервер одним кликом по строке, открыть JSON профиля, проверить сгенерированный sing-box config, удалить сервер, проверить отклик выбранного сервера или всех серверов через Karing-style URL-test и отсортировать таблицу по отклику или качеству.

## Маршрутизация JSON/TXT/SRS

Для JSON поддерживаются структуры с ключами:

- `domains`, `domain`, `domain_suffix`, `domain_keyword`
- `domain_regex`, `regexp`, `regex`
- `geosite`, `geoip`
- `ip`, `ips`, `ip_cidr`, `cidr`
- `process_name`, `process_path`, `process_path_regex`
- `rule_set`, `rule_sets`

Пример:

```json
{
  "domains": ["example.com", "*.example.org"],
  "geosite": ["category-ru"],
  "geoip": ["private"],
  "ip_cidr": ["1.1.1.1/32", "8.8.8.0/24"],
  "process_name": ["telegram.exe"]
}
```

Для TXT поддерживается простой список: один домен, IP/CIDR, geosite/geoip, process-правило или rule-set на строку. Пустые строки и строки с `#`, `;`, `//` пропускаются.

```text
wildberries.ru
wb.ru
*.example.com
geosite:ru
geoip:private
1.1.1.0/24
telegram.exe
process_path:C:\Program Files\App\app.exe
process_path_regex:(?i).*\\chrome\.exe$
```

Также поддерживаются локальные и удаленные sing-box rule-set ресурсы `.srs`. При импорте `.srs` клиент создает `route.rule_set` ресурс и связанный набор маршрутизации, а сам файл не разворачивается в список доменов.

Можно загрузить локальный JSON/TXT/SRS-файл или прямую raw-ссылку, например:

```text
https://raw.githubusercontent.com/communism420/My-Karing-Ruleset/refs/heads/main/griffini.json
https://example.com/rules/ru-sites.srs
```

Для каждого набора в интерфейсе отдельно задается маршрут:

- `Текущий сервер` — совпавший трафик идет через активный серверный профиль.
- `Напрямую` — совпавший трафик идет в обход VPN.

После добавления набора настройте его маршрут прямо в строке таблицы в колонке `Туннелирование`. Приоритеты меняются кнопками `Выше` и `Ниже`: чем выше набор в списке, тем раньше он попадает в runtime-конфиг sing-box. Если включены только правила `Напрямую`, весь остальной трафик идет через текущий сервер. Если включены только правила `Текущий сервер`, весь остальной трафик идет напрямую. При смешанных правилах используется внутренний маршрут по умолчанию.

Встроенный bypass всегда добавляется к runtime-конфигу sing-box и отправляет домены российского «белого списка» напрямую. Пользовательские правила имеют более высокий приоритет, чем встроенный bypass.

## Активность доменов

Вкладка `Активность` показывает свежие домены и поддомены, которые `sing-box` упоминает в runtime-логах. Для каждой записи отображается маршрут: `VPN` или `Напрямую`, правило, количество запросов и время последнего появления.

Фильтр принимает одно или несколько слов: например `google video` покажет только домены, где встречаются оба слова. Из выбранной строки можно сразу создать правило: точный домен или зону домена, с маршрутом `Напрямую` или `Через VPN`. Такие правила сохраняются в агрегированных наборах `Live Activity: напрямую` и `Live Activity: через VPN`.

## Smart Connect и Failover

При ручном подключении клиент не всегда использует выбранный профиль напрямую. Если рядом есть серверы из той же подписки или группы, Smart Connect берёт ограниченный набор кандидатов, проверяет задержку через sing-box Clash API и выбирает лучший сервер с учётом накопленной статистики качества. Это ускоряет подключение к рабочему серверу без полного сканирования всей подписки. В настройках можно выключить `Smart Connect`, тогда ручное подключение будет запускать выбранный сервер напрямую.

Для явного failover выберите сервер во вкладке `Серверы` и нажмите `Failover-группа`. Клиент соберёт группу из соседних серверов той же подписки/группы. Если текущий сервер перестаёт проходить health-check или sing-box падает, приложение пробует следующий живой сервер, записывает результат в локальную статистику и сохраняет порядок работы группы.

Статистика качества хранится локально в пользовательской папке данных. Она включает историю задержек, успешные проверки, ошибки и cooldown-состояния, но не отправляется наружу.

## IPv6, DNS и защита от утечек

В TUN-режиме можно включить IPv6: sing-box получает IPv4 и IPv6 адреса виртуального интерфейса, а DNS/FakeIP config добавляет A/AAAA обработку. В настройках доступны DNS strategy: `Prefer IPv4`, `Prefer IPv6`, `IPv4 only`, `IPv6 only`.

`Kill Switch` в настройках TUN включает `strict_route` в sing-box. Это защищает системный маршрут внутри TUN-конфигурации, но не меняет глобальные правила Windows Firewall.

`Firewall Kill Switch` — отдельный opt-in режим для Windows. Он требует права администратора, меняет DefaultOutboundAction профилей Windows Firewall на `Block` и добавляет allow-правила для sing-box и локального loopback приложения. При отключении клиент пытается восстановить предыдущие значения Firewall из локальной резервной копии. Включайте этот режим только если понимаете, что при ошибке он может оставить систему в fail-closed состоянии до ручного исправления Firewall.

`Proxy guard` включает системный proxy Windows для Proxy-режима. Он не является полноценным firewall kill switch и зависит от того, уважают ли приложения системные proxy-настройки.

## Диагностика ZIP

Во вкладке `Логи` кнопка `Диагностика ZIP` создаёт архив для баг-репорта. В него входят:

- redacted настройки, профили, подписки, правила, Smart Connect stats и failover-группы;
- redacted runtime-config sing-box;
- redacted логи с диска и текущего буфера;
- базовая системная информация;
- manifest и README архива.

Перед записью архив проходит редактирование секретов: URI, UUID, пароли, токены, ключи, адреса серверов, домены, IP и локальные process paths заменяются placeholder-значениями. Диагностика сохраняется локально; приложение не отправляет архив автоматически.

## Автообновление приложения

Во вкладке `О проекте` есть кнопка `Проверить обновления`. Она читает latest release из GitHub Releases проекта, сравнивает версию с `APP_VERSION` и ищет Windows asset с расширением `.exe`, `.msi` или `.zip`. Если включить `Обновления приложения` в настройках, такая проверка выполняется при запуске.

Обновление работает безопасно: клиент скачивает файл в `%USERPROFILE%\Razreshenie VPN\downloads\app-updates` и предлагает открыть installer или папку с файлом. Запущенный exe не заменяется автоматически, потому что Windows блокирует исполняемый файл работающего приложения.

Для release рекомендуется прикладывать Windows asset с понятным именем, например:

```text
Razreshenie-VPN-Client-setup-windows-x64.exe
Razreshenie-VPN-Client-setup-windows-x64.exe.sha256
```

Checksum-файл опционален. Если в release есть `.sha256` или checksum asset, клиент сверит SHA256 скачанного файла.

## FAQ

### Почему TUN требует права администратора?

TUN создаёт виртуальный сетевой интерфейс и системные маршруты Windows. Эти операции требуют UAC. Proxy-режим работает без прав администратора.

### Чем обычный Kill Switch отличается от Firewall Kill Switch?

Обычный `Kill Switch` включает strict route внутри sing-box TUN config. `Firewall Kill Switch` меняет Windows Firewall и блокирует исходящие соединения для профилей Domain/Private/Public, кроме разрешённых правил. Второй вариант сильнее, но требует администратора и аккуратного отключения.

### Почему Smart Connect выбрал не тот сервер, который я выделил?

Выделенный сервер используется как якорь группы. Smart Connect может выбрать соседний сервер из той же подписки/группы, если он быстрее или стабильнее по текущей проверке и истории качества.

### Где хранятся данные?

По умолчанию всё хранится локально в `%USERPROFILE%\Razreshenie VPN`: настройки, профили, подписки, правила, логи, статистика качества, failover-группы, скачанный sing-box и скачанные обновления приложения.

### Что делать, если обновление приложения скачалось как ZIP?

Откройте папку с файлом, распакуйте ZIP и замените старую сборку после закрытия приложения. Автоматическая замена запущенного exe намеренно не выполняется.

### Что делать, если Firewall Kill Switch не отключился?

Запустите приложение от имени администратора и выключите `Firewall Kill Switch` в настройках. Если приложение недоступно, откройте Windows Defender Firewall with Advanced Security и восстановите исходные outbound-политики профилей вручную.

## Сборка в один EXE через PyInstaller

1. Установите зависимости:

```powershell
python -m pip install -r requirements.txt
```

2. Создайте `.ico`:

```powershell
python tools\create_icon.py
```

3. Соберите один EXE без консоли и с иконкой:

```powershell
pyinstaller --noconfirm --clean --onefile --windowed `
  --name "Razreshenie VPN Client 2.0.1" `
  --icon assets\app.ico `
  --version-file tools\windows_version_info.txt `
  --collect-data qfluentwidgets `
  --hidden-import qfluentwidgets `
  --add-data "logo.webp;." `
  --add-data "assets;assets" `
  --distpath exe_release `
  main.py
```

EXE появится в папке `exe_release`.

Сборка выше не требует права администратора при обычном запуске. TUN-режим запросит UAC только при включении или подключении, если прав администратора еще нет.

Перед публикацией release проверьте, что версии согласованы в:

- `utils/version.py`;
- `tools/windows_version_info.txt`;
- имени PyInstaller spec или параметре `--name`;
- названии GitHub Release tag.

Для встроенного updater загрузите в GitHub Release Windows asset `.exe`, `.msi` или `.zip`. Желательно добавить `.sha256` или общий checksum-файл: клиент использует его для проверки скачанного обновления, если найдёт подходящую запись.
