# Smart Connect

Smart Connect выбирает лучший сервер вокруг выбранного профиля перед подключением. Это полезно для подписок, где рядом есть несколько серверов одного региона или группы.

## Где включается

Во вкладке `Настройки` есть переключатель `Smart Connect`. Если он выключен, ручное подключение запускает ровно выбранный сервер.

## Как выбираются кандидаты

Клиент берёт ограниченный набор кандидатов:

- explicit Failover-группу, если выбранный сервер в неё входит;
- серверы той же группы провайдера;
- серверы той же подписки;
- общий список серверов, если другого контекста нет.

Кандидаты проверяются через sing-box Clash API `/proxies/{tag}/delay`. Если быстрая проверка не удалась, используется сохранённая история качества.

## Что влияет на оценку

На score влияют:

- свежая latency;
- EWMA истории задержки;
- успешные и неуспешные подключения;
- подряд идущие ошибки;
- cooldown после падения сервера;
- небольшой бонус стабильным серверам.

Сервер с минимальной итоговой оценкой выбирается для запуска.

## Когда отключать

Smart Connect стоит отключить, если:

- нужно подключаться строго к конкретному серверу;
- провайдер называет разные локации как одну группу;
- подписка содержит серверы с разными правилами доступа;
- вы тестируете один профиль и не хотите автоматической замены.

## Где хранится история качества

История latency, успешных запусков и ошибок хранится локально в папке данных приложения. Она не отправляется разработчикам или провайдеру.

## Практические советы

- После импорта большой подписки запустите проверку всех серверов во вкладке `Серверы`.
- Создайте Failover-группу для важных серверов, если хотите контролировать порядок кандидатов.
- Для цепочек и балансировки используйте [advanced-группы](ADVANCED_GROUPS.md).
- Если Smart Connect часто выбирает нежелательный сервер, проверьте группу провайдера и отключите Smart Connect для ручного теста.
