# Failover и self-healing

Failover отвечает за переключение на следующий рабочий сервер, если текущий сервер перестал отвечать. Self-healing восстанавливает sing-box, если core неожиданно упал.

## Failover-группы

Во вкладке `Серверы` выберите профиль и нажмите `Failover`. Клиент создаст или обновит группу из серверов той же logical-группы:

- explicit группа провайдера, если она есть;
- подписка целиком, если группы нет;
- выбранный сервер как одиночная fallback-группа.

Группы сохраняются локально и используются Smart Connect и автоматическим failover. В редакторе группы можно вручную менять состав и порядок серверов. Этот же редактор используется для advanced-режимов: Multi-hop и Load Balance.

Подробно: [Multi-hop, Load Balance и приоритеты групп](ADVANCED_GROUPS.md).

## Стратегии

Клиент поддерживает несколько подходов:

- `failover_order` - идти по порядку группы после упавшего сервера;
- `latency` - выбирать самый быстрый доступный сервер;
- `smart` - учитывать latency, историю качества и cooldown.

## Health monitor

Фоновая проверка соединения запускается с заданным интервалом, если VPN подключён и приложение не занято другой операцией. После заданного количества ошибок подряд сервис строит план восстановления:

- Failover, если есть активная группа и сервер-якорь;
- restart текущего профиля, если failover невозможен;
- nothing, если восстановление отключено или пользователь отключил VPN вручную.

## Self-healing

Self-healing ограничивает количество автоматических перезапусков. После превышения лимита включается cooldown, чтобы приложение не зациклилось на бесконечном рестарте.

## Что пишется в логи

Критичные события логируются:

- сервер упал, начат failover;
- количество найденных кандидатов;
- ошибки запуска кандидата;
- успешное переключение;
- исчерпание попыток восстановления;
- переход self-healing в cooldown.

## Если failover не сработал

Проверьте:

1. Включён ли `Health monitor` в настройках.
2. Есть ли Failover-группа для текущего сервера.
3. Не было ли ручного отключения VPN.
4. Не находится ли self-healing в cooldown.
5. Есть ли рабочие кандидаты в группе.
