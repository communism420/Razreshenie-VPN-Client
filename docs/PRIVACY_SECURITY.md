# Приватность и безопасность

Razreshenie VPN Client хранит пользовательские данные локально и не содержит телеметрии.

## Где хранятся данные

По умолчанию используется:

```text
%USERPROFILE%\Razreshenie VPN
```

Там находятся:

- настройки;
- профили;
- подписки;
- правила маршрутизации;
- логи;
- статистика качества серверов;
- failover-группы;
- usage-статистика серверов и advanced-групп;
- скачанный sing-box core;
- скачанные обновления приложения.
- временный batch-файл замены EXE, если пользователь выбрал in-place обновление.

## Какие сетевые запросы делает клиент

Клиент обращается в сеть только для пользовательских функций:

- загрузка sing-box release;
- обновление подписок;
- загрузка raw JSON/TXT/SRS правил;
- проверка GitHub Releases приложения;
- проверка публичного IP и DNS;
- health-check подключения;
- проверка задержки серверов через временный sing-box core.

## Диагностика ZIP

Диагностика создаётся локально и не отправляется автоматически. Перед записью архив редактируется:

- URI заменяются placeholder-значениями;
- UUID, пароли, токены и ключи скрываются;
- адреса серверов, домены и IP редактируются;
- локальные process paths скрываются.

## Kill Switch

Обычный `Kill Switch` включает `strict_route` в TUN config sing-box.

`Firewall Kill Switch` меняет Windows Firewall DefaultOutboundAction на `Block` и добавляет allow-правила для sing-box и loopback. Это более строгий режим, но он требует администратора и может оставить систему в fail-closed состоянии при ошибке. Включайте его только осознанно.

## Updater

Updater скачивает release asset в локальную папку и предлагает открыть файл или папку. Он не заменяет запущенный EXE автоматически.

## Что не делает клиент

- Не отправляет телеметрию.
- Не отправляет диагностику автоматически.
- Не передаёт подписки или профили разработчику.
- Не изменяет удалённый репозиторий.
