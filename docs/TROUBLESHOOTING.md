# Troubleshooting

Этот документ описывает безопасные шаги диагностики. Он не требует отправлять данные наружу.

## Сайт не открывается

1. Проверьте, подключён ли VPN и выбран ли ожидаемый режим.
2. Откройте вкладку `Активность` и посмотрите, каким маршрутом идут домены сайта и CDN.
3. Если сайт в bypass, добавьте также домены плеера, API и CDN.
4. Если проблема только в одном приложении, создайте per-app правило.
5. В TUN проверьте DNS strategy и IPv6.

## Плеер на сайте не работает

Часто HTML сайта и видео идут с разных доменов. Добавьте правило не только для основного домена, но и для доменов плеера, API, статических файлов и CDN. Live Activity помогает увидеть эти домены во время открытия страницы.

## Wildberries, Ozon или российские сайты не открываются

Проверьте:

- включён ли встроенный bypass;
- нет ли пользовательского правила выше bypass, которое отправляет эти домены через VPN;
- не включён ли слишком строгий Firewall Kill Switch;
- не ломает ли IPv6 маршрут провайдера.

Если сайт работает без VPN, но не работает в TUN с bypass, временно создайте точное правило `Напрямую` выше остальных правил.

## TUN не стартует

TUN требует прав администратора. Запустите приложение от имени администратора или подтвердите UAC при подключении. Проверьте, что sing-box core скачан и не заблокирован антивирусом.

## Proxy работает, но приложения не используют VPN

Proxy-режим создаёт локальный SOCKS5/HTTP inbound. Не все приложения используют системный proxy. Для таких приложений используйте TUN или настройте proxy внутри самого приложения.

## Smart Connect выбрал другой сервер

Это нормальное поведение, если Smart Connect включён. Он выбирает лучший сервер из группы по latency и истории качества. Для строгого выбора отключите Smart Connect в настройках.

## Failover не переключает сервер

Проверьте:

- есть ли Failover-группа;
- включён ли Health monitor;
- не отключал ли пользователь VPN вручную;
- есть ли доступные кандидаты;
- не исчерпан ли self-healing лимит.

## Firewall Kill Switch не отключился

1. Запустите приложение от имени администратора.
2. Выключите `Firewall Kill Switch` в настройках.
3. Если приложение не запускается, откройте Windows Defender Firewall with Advanced Security и восстановите исходные outbound policies вручную.

## Подписка импортируется частично

Клиент пробует основной запрос, retry без conditional headers, retry с no-cache и last-good cache. Если результат всё равно неполный, проверьте URL подписки в браузере и лимиты провайдера.

## Экспорт диагностики

Во вкладке `Логи` нажмите `Диагностика ZIP`. Архив создаётся локально и содержит redacted настройки, runtime config, логи и системную информацию. URI, ключи, домены, IP и локальные пути заменяются placeholder-значениями.
